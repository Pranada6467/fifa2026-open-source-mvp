## Problem Statement

Build a FIFA World Cup 2026 prediction system as a vehicle for learning data/ML
work hands-on, and good enough to show off publicly. The system rates team
strength from historical international results, converts that into per-match
W/D/L and scoreline probabilities, simulates the full 2026 tournament bracket
(48 teams, 12 groups, best-third-place routing) via Monte Carlo, and — crucially
— runs as a **live loop**: after every result it re-scores its own past
predictions, recalibrates ratings, ingests fresh data and bookmaker odds, and
republishes. The tournament is live (opens 2026-06-11), so the system runs
continuously across the five weeks. Model quality leads; the calendar is context,
not a cap on what the model is allowed to do.

## What Makes This Cool

The "whoa" is not flashy UI — it's **honest calibration in public**. The product
is a scoreboard that proves "when I say 70%, it happens ~70% of the time," plus a
**technique leaderboard** where every method (Elo, Dixon-Coles, time-decay,
market-blend, agentic panel) is ranked by out-of-sample log-loss / Brier. The
live World Cup is the dramatic public feed; thousands of historical matches are
the private proving ground. The curiosity engine: every new technique is an
experiment with a verdict, so "what more can I do" always has a measurable
answer.

## Constraints

- The tournament is live (opens 2026-06-11, group stage ~Jun 11–27). This is
  context that makes a continuous system valuable — **not** a deadline that caps
  model scope. Build the model you actually want; let it improve in-flight.
- Must be **live and continuous**: recalibrate after every match, ingest latest
  warm-up/qualifier/tournament results and odds as they update. Not a one-shot
  static build.
- Solo builder, final-year AI/ML student, comfortable with Python/pandas/numpy/
  scikit-learn and ML concepts, builds agentic workflows. Pairing with Claude
  to accelerate.
- "Show it off" → must be public and live (a private notebook fails the goal).
- No paid data subscription assumed for v1; free/open data + free odds tier.

## Premises (confirmed)

1. The spine (Elo + `penaltyblog` Poisson/Dixon-Coles + sklearn metrics) is fast
   to assemble. The real work and the real learning are the **custom 2026
   simulator** and the **validation methodology**, not the compilation.
2. Calibration is *proven* on history (thousands of matches) and *demonstrated*
   live (the small, dramatic public sample, ~104 matches). Two tracks — don't
   conflate them.
3. The betting market is the baseline and is already near-perfectly calibrated.
   Win = "well-calibrated, close to market." Measure against odds; don't try to
   beat them in v1.
4. Live data is unrepeatable, so the loop's plumbing (predict → log → score →
   recalibrate) should run end-to-end early — capturing the opener is a free bonus
   if the model is ready, never a reason to ship a worse model. Sequencing, not a
   quality cap.
5. "Show it off" means public + live, with a daily update ritual.

**Builder's refinement (accepted, now central):** the project IS the live update
loop. `predict → log → result → re-score → recalibrate → republish`, repeated
after every match. The warm-up friendlies happening now are the first fuel.

## Approaches Considered

### Approach A: Loop first, simulator later (minimal viable)
Full live loop with the simplest model: Elo from martj42 history + current
warm-ups → `penaltyblog` Dixon-Coles per-match W/D/L + scoreline grid → log
timestamped predictions → after each result re-score calibration + recompute
Elo → thin Streamlit board. No tournament simulator yet. Effort S–M, Risk Low.
Captures the opener and every group match from game 1.

### Approach B: Full spine + Monte Carlo simulator before opener (ideal)
Everything in A plus the custom 2026 Monte Carlo simulator (group → best-third
routing → knockouts → winner probabilities) and a 2014/18/22 backtest, all
before the opener. Complete from day one. Risk: the simulator + best-third
routing + backtest is the fiddliest piece; trying to finish it in the window
risks shipping nothing live by kickoff. Effort XL, Risk Med–High for the
deadline.

### Approach C: Calibration Lab / experiment harness (creative)
Structure everything as a model registry + scoring harness from the start. Each
model is a pluggable object (Elo, Dixon-Coles, TimeDecayElo, MarketBlend, later
an agentic adjustment panel); one scorer ranks them all by out-of-sample
log-loss/Brier. The "product" is the technique leaderboard. Max learning-per-hour,
fits the agentic-workflow background. Effort L, Risk Med (upfront abstraction).

## Recommended Approach

**Chosen: B + C fused, full quality.** The builder chose the ambitious line —
Monte Carlo required, no compromise on model sophistication. Build order is driven
by good engineering, not by the calendar: stand up the live loop's plumbing
(predict → log → score → recalibrate) end-to-end first because that validates the
hardest integration risk and lets every later technique be measured immediately.
The simulator reads the same ratings the per-match predictor produces — it is
additive, not a rewrite — so it layers on top without rework.

Concretely: C's pluggable-model seam is the architecture from line one; the loop
is the first thing that runs end-to-end; B's simulator layers in next. The model
can be as rich as you want at every step — the sequencing only governs *what runs
first*, never *how good it is allowed to be*.

### The technique ladder (the curiosity fuel — each rung is a measurable experiment)
1. Baseline Elo → home/neutral-adjusted goal expectation → Poisson W/D/L.
2. Dixon-Coles correction (fixes the low-score draw bias Poisson gets wrong).
3. Time-decay weighting (recent matches count more) + match-importance weights
   (WC > continental > qualifier > Nations League > friendly).
4. Separate attack/defence ratings (bivariate Poisson) instead of single Elo gap.
5. Market blend: pull bookmaker odds, de-vig with `penaltyblog`, blend model +
   market, measure whether the blend's log-loss beats either alone.
6. xG-adjusted form (later, if a free xG source is reachable) — don't overreact
   to lucky scorelines.
7. Agentic adjustment panel (your wheelhouse): bounded agents propose small
   Elo/xG nudges with reasons; **only kept if they improve backtest log-loss,
   else disabled.** A tracked experiment, not a gimmick.

Every rung is added to the registry and ranked by the same scorer. That's the
whole point: you never guess whether you improved — the number tells you.

## Open Questions

1. **2026 best-third-place routing matrix (highest priority correctness item).**
   The 48-team format sends 8 of 12 third-placed teams to the Round of 32, and
   which group winner faces which third depends on *which* groups those thirds
   come from. This is the official FIFA bracket mapping (the "Annex C"–style
   table). Must be transcribed from the FIFA source of truth and unit-tested
   before any winner probability is trusted. A bug here is silent.
2. **Data freshness for current warm-ups.** martj42/international_results is
   community-maintained; confirm how current it is for June 2026 friendlies, and
   build the manual-add path so you can enter a result the dataset is missing.
3. **Odds source.** The Odds API free tier (h2h + outrights) vs scraping. Pick
   one and snapshot odds with timestamps from day one (you can't backfill them).
4. **Hosting.** Streamlit Community Cloud (free, GitHub-connected) is the default
   for "public + live." Confirm it can run the daily update (or run the update
   locally/GitHub Action and push results the app reads).
5. **ET/penalties model.** Knockout draws after 90' → ET then penalties. v1:
   approximate penalties as near coin-flip with a small favourite edge; revisit
   later.

## Success Criteria

- The loop runs end-to-end: per-match predictions logged with timestamps to an
  append-only store, and a calibration scorer that re-scores past predictions and
  recomputes ratings when a new result lands, without manual surgery. (Capturing
  the opener is a nice-to-have, not a pass/fail gate.)
- Monte Carlo simulator produces group-qualification, knockout, and winner
  probabilities, with the best-third routing unit-tested against the official
  bracket.
- A reliability/calibration curve and a running log-loss/Brier number are visible
  on a public Streamlit page.
- At least 3 techniques registered and ranked by out-of-sample score.
- Backtest on 2014/18/22 shows the model is reasonably calibrated (not necessarily
  market-beating).

## Distribution Plan

- **Code:** GitHub repo (`git init` first — this is not a repo yet).
- **Public surface:** Streamlit Community Cloud, connected to the repo (free,
  redeploys on push).
- **Update cadence:** daily manual run before each match day in v1; promote to a
  scheduled GitHub Action later. Predictions and odds snapshots are committed/
  stored append-only so calibration history is reproducible.

## Next Steps (ordered by engineering dependency, not by the calendar)

**Block 0 — the live loop runs end-to-end:**
1. `git init`, project skeleton, `requirements.txt` (pandas, numpy, scipy,
   scikit-learn, penaltyblog, streamlit, requests).
2. Build a `Model` interface (the C seam) with one method: `predict(home, away,
   neutral) -> {p_home, p_draw, p_away, score_grid}`.
3. Implement `BaselineElo` + Poisson/Dixon-Coles via `penaltyblog`, fit on
   martj42 history. Add the manual-result entry path.
4. `predictions_log` (append-only CSV/SQLite): match, model, probs, timestamp.
5. `score.py`: when a result lands, compute per-prediction log-loss/Brier, append
   to the calibration history.
6. Minimal Streamlit page: today's match predictions + running calibration curve.
   Deploy it. (If it's live in time, it starts logging the opener — bonus, not a
   gate.)

**Block 1 — the simulator:**
7. Group-stage table logic + tiebreakers from FIFA rules.
8. **Transcribe and unit-test the best-third routing matrix** (open question #1).
9. Monte Carlo: simulate group → R32 → … → final N times; output stage + winner
   probabilities. ET/penalties approximation.
10. Backtest harness on 2014/18/22; report calibration.

**Block 2 — climb the technique ladder:**
11. Add odds ingestion + de-vig + market blend (rung 5), snapshot odds daily.
12. Add time-decay + importance weights (rung 3), measure deltas.
13. Later rungs (xG, agentic panel) as tracked experiments.

Build the model you actually want at each step. Sequencing governs what runs
first, never how good it's allowed to be.
